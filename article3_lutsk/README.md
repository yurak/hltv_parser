# Стаття 3 (подання в Луцьк) — Інваріантність ознак CS2/CSGO

**Третя стаття** серійної публікаційної лінії. Відтворюваний робочий процес для статті про
інваріантність метрик продуктивності гравців Counter-Strike (CS2 vs CSGO) за вимірами:
карти, версії гри, сторони (T/CT).

Цільове видання: **№85 «Комп'ютерно-інтегровані технології» (ЛНТУ, Луцьк)**, категорія «Б»,
4 випуски/рік. Найближчий дедлайн подання — **20.08.2026** (вересневий випуск).
Повні вимоги та посилання на журнал — `paper/journal_requirements.md`.

## Основні етапи

1. Витяг вимог журналу (`/journal-requirements`)
2. Інспекція даних та план аналізу (`/analysis-plan`, plan mode)
3. Відтворювані скрипти аналізу (`/analyze-data`)
4. Таблиці та фігури (`/build-figures`)
5. Чернетка рукопису (`/draft-paper`)
6. Незалежна рецензія (`/reviewer`)
7. Ревізії та revision log (`/revise`)
8. Експорт DOCX і пакет для керівника (`/export-docx`, `/supervisor-package`)

## Важливі шляхи

- `data/raw/` — оригінальні дані (2 файли map_stats CS2/CSGO), не редагувати.
- `data/processed/` — очищені дані.
- `scripts/` — аналіз/експорт.
- `outputs/tables/`, `outputs/figures/`, `outputs/logs/` — згенеровані артефакти.
- `paper/` — рукопис і матеріали статті.
- `review/` — коментарі рецензента й логи.
- `references/` — настанови журналу, джерела, бібліографія.

## Ключове правило

Не писати результати в рукописі, доки не існує відповідних вихідних артефактів аналізу.
